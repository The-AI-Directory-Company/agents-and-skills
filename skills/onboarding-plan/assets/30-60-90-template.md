# 30-60-90 Day Template

A bare milestone table schema for any onboarding context — new hire, new customer, or new user. Fill in persona-specific activities and success criteria based on the role or use case.

## Instructions

1. Define the onboarding context in the Overview section.
2. Fill in activities and success criteria for each milestone. Every row must have a measurable success criterion — not just a task description.
3. Schedule feedback checkpoints at each milestone boundary.
4. Adjust the timeline if needed (some roles ramp in 60 days, some in 120), but maintain the milestone structure.

---

## Overview

```
Onboardee:         [Name / role / persona]
Team / Account:    [Team name or customer account]
Timeline:          [30 / 60 / 90 / 120 days]
Onboarding owner:  [Manager / buddy / CSM / other]
Success measure:   [What "fully ramped" looks like — concrete and measurable]
Start date:        YYYY-MM-DD
```

## Feedback Schedule

| Timing | Format | Participants | Focus |
|--------|--------|-------------|-------|
| Daily (Week 1) | 15-min sync | Onboardee + buddy/owner | "What's blocking you?" |
| Weekly (Weeks 2-4) | 30-min 1:1 | Onboardee + owner | Progress against Milestone 1 |
| Day 30 | Written + meeting | Onboardee + owner | Milestone 1 review, bidirectional feedback |
| Biweekly (Weeks 5-8) | 30-min 1:1 | Onboardee + owner | Progress against Milestone 2 |
| Day 60 | Written + meeting | Onboardee + owner | Milestone 2 review, bidirectional feedback |
| Biweekly (Weeks 9-12) | 30-min 1:1 | Onboardee + owner | Progress against Milestone 3 |
| Day 90 | Formal review | Onboardee + owner + skip-level (if applicable) | Final assessment, transition to BAU |
